## 2. Schema Descriptions

### Mission

**Purpose:** Stores space exploration missions with their destination, timeline, status, and objective.

| Attribute      | Type     | Required | Constraints                                         |
|----------------|----------|----------|-----------------------------------------------------|
| _id            | ObjectId | auto     | Primary key, auto-generated by MongoDB              |
| name           | String   | yes      | Trimmed, non-empty                                  |
| destination    | String   | yes      | Trimmed, e.g. Moon, Mars, Europa, Titan             |
| launchDate     | Date     | yes      | Planned launch date                                 |
| status         | String   | yes      | Enum: `Planning`, `Preparing`, `Traveling`, `Exploring`, `PreparingReturn`, `Returning`, `Completed`, `Failed` |
| phaseStartedAt | Date     | no       | Tracks when the current phase began (null by default, set on transition) |
| rocketType     | String   | yes      | Enum: `falcon9`, `shuttle`, `saturnV` (default: `falcon9`) |
| description    | String   | no       | Trimmed, free-text mission objective                |
| createdAt      | Date     | auto     | Mongoose timestamp                                  |
| updatedAt      | Date     | auto     | Mongoose timestamp                                  |

**DAO Methods (Mongoose):**

| Method                          | Description                          |
|---------------------------------|--------------------------------------|
| `Mission.find()`               | Retrieve all missions                |
| `Mission.findById(id)`         | Retrieve a single mission by ID      |
| `Mission.create(data)`         | Create a new mission                 |
| `Mission.findByIdAndUpdate(id, data)` | Update an existing mission     |
| `Mission.findByIdAndDelete(id)`| Delete a mission by ID               |

### CrewMember

**Purpose:** Stores astronaut crew members with their role, background, and mission assignment.

| Attribute       | Type     | Required | Constraints                                                  |
|-----------------|----------|----------|--------------------------------------------------------------|
| _id             | ObjectId | auto     | Primary key, auto-generated by MongoDB                       |
| name            | String   | yes      | Trimmed, non-empty                                           |
| role            | String   | yes      | Enum: `Commander`, `Pilot`, `Engineer`, `Scientist`, `Medic` |
| nationality     | String   | yes      | Trimmed, country of origin                                   |
| experienceLevel | String   | yes      | Enum: `Rookie`, `Veteran`, `Elite`                           |
| missionId       | ObjectId | no       | FK вҶ’ Mission._id, null if unassigned                         |
| createdAt       | Date     | auto     | Mongoose timestamp                                           |
| updatedAt       | Date     | auto     | Mongoose timestamp                                           |

**DAO Methods (Mongoose):**

| Method                                | Description                            |
|---------------------------------------|----------------------------------------|
| `CrewMember.find()`                  | Retrieve all crew members              |
| `CrewMember.findById(id)`           | Retrieve a single crew member by ID    |
| `CrewMember.find({ missionId: id })` | Retrieve crew members for a mission    |
| `CrewMember.create(data)`           | Create a new crew member               |
| `CrewMember.findByIdAndUpdate(id, data)` | Update an existing crew member    |
| `CrewMember.findByIdAndDelete(id)`  | Delete a crew member by ID             |
| `CrewMember.updateMany(filter, data)`| Bulk update (e.g. unassign crew)       |

## 3. API Endpoints (Commands)

### Mission Endpoints

#### GET /api/missions вҖ” List All Missions

- **Description:** Retrieves all missions sorted by creation date (newest first)
- **Request body:** None
- **Response:** `200` вҖ” Array of mission objects

#### GET /api/missions/:id вҖ” Get Mission

- **Description:** Retrieves a single mission by its ID
- **Request body:** None
- **Response:** `200` вҖ” Mission object | `404` вҖ” Mission not found

#### POST /api/missions вҖ” Create Mission

- **Description:** Creates a new space mission
- **Request body:**
  ```json
  {
    "name": "Mars Odyssey",
    "destination": "Mars",
    "launchDate": "2027-06-15",
    "status": "Planning",
    "rocketType": "falcon9",
    "description": "First crewed mission to Mars"
  }
  ```
- **Response:** `201` вҖ” Created mission | `400` вҖ” Validation error

#### PUT /api/missions/:id вҖ” Update Mission

- **Description:** Updates an existing mission (including status changes)
- **Request body:** Any subset of mission fields
- **Response:** `200` вҖ” Updated mission | `404` вҖ” Not found | `400` вҖ” Validation error

#### DELETE /api/missions/:id вҖ” Delete Mission

- **Description:** Deletes a mission and unassigns all its crew members (sets their missionId to null)
- **Request body:** None
- **Response:** `200` вҖ” `{ "message": "Mission deleted" }` | `404` вҖ” Not found

#### PATCH /api/missions/:id/transition вҖ” Transition Mission Status

- **Description:** Validates and transitions a mission's status to a new phase, setting phaseStartedAt to the current timestamp
- **Request body:**
  ```json
  {
    "status": "Preparing"
  }
  ```
- **Valid transitions:**
  | From             | Allowed targets                |
  |------------------|--------------------------------|
  | Planning         | Preparing (requires crew)      |
  | Preparing        | Traveling, Failed              |
  | Traveling        | Exploring, Failed              |
  | Exploring        | PreparingReturn, Failed        |
  | PreparingReturn  | Returning, Failed              |
  | Returning        | Completed, Failed              |
  | Completed        | Planning                       |
  | Failed           | Planning                       |
- **Response:** `200` вҖ” Updated mission | `400` вҖ” Invalid transition or no crew assigned | `404` вҖ” Not found

#### GET /api/missions/:id/crew вҖ” Get Mission Crew

- **Description:** Retrieves all crew members assigned to a specific mission
- **Request body:** None
- **Response:** `200` вҖ” Array of crew member objects | `404` вҖ” Mission not found

### CrewMember Endpoints

#### GET /api/crew-members вҖ” List All Crew Members

- **Description:** Retrieves all crew members with populated mission info, sorted by creation date (newest first)
- **Request body:** None
- **Response:** `200` вҖ” Array of crew member objects (with mission name & destination populated)

#### GET /api/crew-members/:id вҖ” Get Crew Member

- **Description:** Retrieves a single crew member by ID with populated mission info
- **Request body:** None
- **Response:** `200` вҖ” Crew member object | `404` вҖ” Not found

#### POST /api/crew-members вҖ” Create Crew Member

- **Description:** Creates a new crew member (initially unassigned unless missionId is provided)
- **Request body:**
  ```json
  {
    "name": "Elena Kovacs",
    "role": "Scientist",
    "nationality": "Hungarian",
    "experienceLevel": "Veteran",
    "missionId": null
  }
  ```
- **Response:** `201` вҖ” Created crew member | `400` вҖ” Validation error

#### PUT /api/crew-members/:id вҖ” Update Crew Member

- **Description:** Updates an existing crew member (including mission assignment/reassignment)
- **Request body:** Any subset of crew member fields
- **Response:** `200` вҖ” Updated crew member | `404` вҖ” Not found | `400` вҖ” Validation error

#### DELETE /api/crew-members/:id вҖ” Delete Crew Member

- **Description:** Deletes a crew member
- **Request body:** None
- **Response:** `200` вҖ” `{ "message": "Crew member deleted" }` | `404` вҖ” Not found

```bash
cd server
cp .env.example .env
# Edit .env with your MongoDB Atlas connection string
npm install
npm run dev
```

Server starts at `http://localhost:5000`

## 6. Seed Data

The server includes a seed script (`server/seed.js`) that auto-populates the database on first run when no missions exist.

- **6 missions:** Artemis VII (Moon), Ares Pioneer (Mars), Aphrodite Survey (Venus), Europa Deep Dive (Europa), Titan Frontier (Titan), Kronos Ring Run (Saturn)
- **12 crew members:** 6 pre-assigned to missions (Artemis VII, Ares Pioneer, Europa Deep Dive), 6 unassigned and available
- **Trigger:** Runs automatically at server startup if `Mission.countDocuments()` returns 0
- **Idempotent:** Skips seeding if any missions already exist
